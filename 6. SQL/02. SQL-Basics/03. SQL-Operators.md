# üìò **SQL Operators**

## üß© **Introduction**

**SQL operators** are special symbols or keywords used to perform **operations on data** stored in database tables.
They help you **filter**, **compare**, **combine**, or **manipulate** data in SQL queries.

Operators are used in SQL statements‚Äîespecially in the `WHERE`, `SELECT`, and `HAVING` clauses‚Äîto form conditions or expressions.

---

## üî¢ **Types of SQL Operators**

SQL operators are broadly classified into:

1. **Arithmetic Operators**
2. **Comparison (Relational) Operators**
3. **Logical Operators**
4. **Bitwise Operators**
5. **Assignment Operators**
6. **Set Operators**
7. **Special Operators**

---

## ‚ûï **1. Arithmetic Operators**

Used for performing **mathematical calculations** on numeric data.

| Operator | Description         | Example          | Result |
| -------- | ------------------- | ---------------- | ------ |
| `+`      | Addition            | `SELECT 10 + 5;` | 15     |
| `-`      | Subtraction         | `SELECT 10 - 5;` | 5      |
| `*`      | Multiplication      | `SELECT 10 * 5;` | 50     |
| `/`      | Division            | `SELECT 10 / 5;` | 2      |
| `%`      | Modulus (remainder) | `SELECT 10 % 3;` | 1      |

üìò **Use Case:**

```sql
SELECT product_name, price, price * 0.1 AS discount
FROM products;
```

‚û° Calculates a 10% discount on each product.

---

## ‚öñÔ∏è **2. Comparison (Relational) Operators**

Used to **compare two values**.
They return a **boolean result**: `TRUE`, `FALSE`, or `UNKNOWN` (when involving `NULL`).

| Operator    | Description           | Example           | Result             |
| ----------- | --------------------- | ----------------- | ------------------ |
| `=`         | Equal to              | `age = 25`        | True if age is 25  |
| `!=` / `<>` | Not equal to          | `age <> 25`       | True if age not 25 |
| `>`         | Greater than          | `salary > 50000`  | True if more       |
| `<`         | Less than             | `marks < 50`      | True if below 50   |
| `>=`        | Greater than or equal | `salary >= 40000` | True if ‚â• 40000    |
| `<=`        | Less than or equal    | `marks <= 90`     | True if ‚â§ 90       |

üìò **Use Case:**

```sql
SELECT * FROM employees
WHERE salary > 60000;
```

‚û° Retrieves employees earning more than ‚Çπ60,000.

---

## üß† **3. Logical Operators**

Used to **combine multiple conditions** in SQL statements.

| Operator | Description                             | Example                           | Result           |
| -------- | --------------------------------------- | --------------------------------- | ---------------- |
| `AND`    | Returns TRUE if all conditions are TRUE | `age > 18 AND city = 'Mumbai'`    | Both must match  |
| `OR`     | Returns TRUE if any condition is TRUE   | `city = 'Delhi' OR city = 'Pune'` | Either matches   |
| `NOT`    | Reverses logical state                  | `NOT (age < 18)`                  | True if age ‚â• 18 |

üìò **Use Case:**

```sql
SELECT * FROM students
WHERE marks >= 75 AND course = 'Computer Science';
```

‚û° Selects students who scored ‚â• 75 and study Computer Science.

---

## ‚öôÔ∏è **4. Bitwise Operators**

Operate on binary representations of numbers (bit-level operations).
Used rarely in general SQL queries.

| Operator | Description | Example    | Result |     |     |
| -------- | ----------- | ---------- | ------ | --- | --- |
| `&`      | Bitwise AND | `5 & 3`    | 1      |     |     |
| `        | `           | Bitwise OR | `5     | 3`  | 7   |
| `^`      | Bitwise XOR | `5 ^ 3`    | 6      |     |     |
| `~`      | Bitwise NOT | `~5`       | -6     |     |     |
| `<<`     | Left shift  | `5 << 1`   | 10     |     |     |
| `>>`     | Right shift | `5 >> 1`   | 2      |     |     |

üìò Example:

```sql
SELECT 12 & 5;  -- returns 4
```

---

## üßæ **5. Assignment Operator**

Used to **assign a value** to a variable (mainly in procedural SQL, like T-SQL or PL/SQL).

| Operator | Description   | Example                          |
| -------- | ------------- | -------------------------------- |
| `=`      | Assigns value | `SET @total = price * quantity;` |

üìò **Use Case (MySQL):**

```sql

SET @y = 10;

```

---

## üßÆ **6. Set Operators**

Used to **combine the results of two or more SELECT queries**.
Each SELECT must have the same number of columns and compatible data types.

| Operator           | Description                                  | Example                                                            |
| ------------------ | -------------------------------------------- | ------------------------------------------------------------------ |
| `UNION`            | Combines results, removes duplicates         | `SELECT city FROM customers UNION SELECT city FROM suppliers;`     |
| `UNION ALL`        | Combines results, includes duplicates        | `SELECT city FROM customers UNION ALL SELECT city FROM suppliers;` |
| `INTERSECT`        | Returns common records                       | `SELECT city FROM table1 INTERSECT SELECT city FROM table2;`       |
| `EXCEPT` / `MINUS` | Returns records in first query not in second | `SELECT city FROM table1 EXCEPT SELECT city FROM table2;`          |

üìò **Use Case:**

```sql
SELECT name FROM students_2024
UNION
SELECT name FROM students_2025;
```

‚û° Combines names of students from both years.

## üß© **7. Special Operators**

SQL provides special operators for **pattern matching**, **range checking**, and **NULL handling**.

| Operator          | Description                       | Example                                             | Meaning                           |
| ----------------- | --------------------------------- | --------------------------------------------------- | --------------------------------- |
| `BETWEEN ... AND` | Checks if value is within range   | `marks BETWEEN 50 AND 80`                           | True if 50 ‚â§ marks ‚â§ 80           |
| `IN (list)`       | Matches any value in list         | `city IN ('Delhi', 'Pune', 'Chennai')`              | True if city matches one          |
| `LIKE`            | Pattern match using wildcards     | `name LIKE 'R%'`                                    | Names starting with R             |
| `IS NULL`         | Checks if value is NULL           | `address IS NULL`                                   | True if no address                |
| `IS NOT NULL`     | Checks if not NULL                | `address IS NOT NULL`                               | True if value exists              |
| `EXISTS`          | Checks if subquery returns rows   | `EXISTS (SELECT * FROM orders WHERE customer_id=5)` | True if result found              |
| `ANY`             | Compares value to any in subquery | `salary > ANY (SELECT salary FROM employees)`       | True if greater than at least one |
| `ALL`             | Compares value to all in subquery | `salary > ALL (SELECT salary FROM employees)`       | True if greater than all          |

üìò **Example:**

```sql
SELECT * FROM employees
WHERE department IN ('IT', 'HR')
  AND salary BETWEEN 30000 AND 80000;
```

## üí° **Operator Precedence (Execution Order)**

When multiple operators appear in one expression, SQL follows **operator precedence** (priority order):

| Precedence | Operator Category                |
| ---------- | -------------------------------- |
| 1Ô∏è‚É£         | Arithmetic (`*`, `/`, `%`)       |
| 2Ô∏è‚É£         | Arithmetic (`+`, `-`)            |
| 3Ô∏è‚É£         | Comparison (`=`, `<`, `>`, etc.) |
| 4Ô∏è‚É£         | Logical (`NOT`)                  |
| 5Ô∏è‚É£         | Logical (`AND`)                  |
| 6Ô∏è‚É£         | Logical (`OR`)                   |

üìò Example:

```sql

WHERE department = 'IT' OR salary > 60000 AND experience > 5;
```

‚û° `AND` executes before `OR` ‚Äî parentheses can override this:

```sql

```

## üß± **Best Practices for Using Operators**

1. ‚úÖ Use **parentheses** to make logic clear.
2. ‚öôÔ∏è Avoid unnecessary **type conversion** (e.g., comparing string to number).
3. üìä Use `IN` instead of multiple `OR` conditions for cleaner queries.
4. üß© Use `BETWEEN` only for **inclusive ranges**.
5. üïµÔ∏è Use `IS NULL` instead of `= NULL` (since `NULL` ‚â† anything).
