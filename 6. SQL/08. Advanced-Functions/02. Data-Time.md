# üìò **SQL Date and Time Functions**

SQL **Date and Time functions** allow you to store, retrieve, and manipulate temporal (time-related) data.
These functions are crucial for operations like filtering by date, calculating durations, formatting timestamps, or extracting date parts.

SQL databases like **MySQL**, **PostgreSQL**, and **SQL Server** all support these functions (some with slightly different syntax).

---

## üïí **1. DATE Data Type**

**Definition:**
`DATE` stores **calendar dates** ‚Äî year, month, and day.

**Format:**
`YYYY-MM-DD`

**Example:**

```sql
CREATE TABLE events (
  event_id INT,
  event_date DATE
);
```

**Sample Value:**
`'2025-10-10'`

**Common Operations:**

```sql
-- Get current date
SELECT CURRENT_DATE;           -- or GETDATE(), depending on DBMS

-- Extract year/month/day
SELECT YEAR(event_date), MONTH(event_date), DAY(event_date)
FROM events;

-- Compare dates
SELECT * FROM events WHERE event_date = '2025-10-10';
```

---

## ‚è∞ **2. TIME Data Type**

**Definition:**
`TIME` stores **time of day** (hours, minutes, seconds).

**Format:**
`HH:MI:SS`

**Example:**

```sql
CREATE TABLE schedule (
  start_time TIME,
  end_time TIME
);
```

**Sample Value:**
`'14:30:00'`

**Common Functions:**

```sql
-- Get current system time
SELECT CURRENT_TIME;

-- Extract hour/minute/second
SELECT HOUR(start_time), MINUTE(start_time), SECOND(start_time)
FROM schedule;
```

---

## üïì **3. TIMESTAMP Data Type**

**Definition:**
`TIMESTAMP` stores both **date and time** (optionally with time zone).

**Format:**
`YYYY-MM-DD HH:MI:SS`

**Example:**

```sql
CREATE TABLE logs (
  log_id INT,
  log_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**Sample Value:**
`'2025-10-10 14:35:00'`

**Common Operations:**

```sql
-- Get current timestamp
SELECT CURRENT_TIMESTAMP;

-- Add or subtract interval
SELECT log_time + INTERVAL '1 DAY' FROM logs;
```

---

## üìÜ **4. DATETIME (MySQL / SQL Server Specific)**

**Definition:**
`DATETIME` is similar to `TIMESTAMP` but doesn‚Äôt automatically track timezone or auto-update.

**Example:**

```sql
CREATE TABLE appointments (
  appointment_time DATETIME
);
```

**Common Usage:**

```sql
INSERT INTO appointments VALUES ('2025-10-10 09:30:00');
SELECT NOW();         -- Current datetime
```

---

## üìÖ **5. DATEPART() Function**

**Purpose:**
Extracts a specific **part** (year, month, day, hour, etc.) from a date or time value.

**Syntax (SQL Server):**

```sql
DATEPART(part, date)
```

**Example:**

```sql
SELECT DATEPART(year, '2025-10-10') AS YearPart;
SELECT DATEPART(month, '2025-10-10') AS MonthPart;
SELECT DATEPART(day, '2025-10-10') AS DayPart;
```

**Result:**

| YearPart | MonthPart | DayPart |
| -------- | --------- | ------- |
| 2025     | 10        | 10      |

**Common Parts:**

-   `year`, `quarter`, `month`, `day`, `week`, `weekday`
-   `hour`, `minute`, `second`

---

## ‚ûï **6. DATEADD() Function**

**Purpose:**
Adds (or subtracts) a specific interval (like days, months, years) to a date.

**Syntax (SQL Server):**

```sql
DATEADD(interval, number, date)
```

**Example:**

```sql
SELECT DATEADD(day, 7, '2025-10-10') AS NextWeek;     -- Add 7 days
SELECT DATEADD(month, -2, '2025-10-10') AS TwoMonthsAgo;  -- Subtract 2 months
```

**Result:**

| NextWeek   | TwoMonthsAgo |
| ---------- | ------------ |
| 2025-10-17 | 2025-08-10   |

---

## üìâ **7. DATEDIFF() Function**

**Purpose:**
Calculates the **difference between two dates** in a given unit (days, months, years, etc.).

**Syntax:**

```sql
DATEDIFF(interval, start_date, end_date)
```

**Example:**

```sql
SELECT DATEDIFF(day, '2025-10-01', '2025-10-10') AS DaysDiff;
```

**Result:**
`9`

---

## üìÖ **8. FORMAT() Function**

**Purpose:**
Formats a date/time value into a specific string format.

**Example:**

```sql
SELECT FORMAT('2025-10-10', 'dd/MM/yyyy') AS FormattedDate;
```

**Result:**
`10/10/2025`

---

## üß© **9. GETDATE() / NOW()**

**Purpose:**
Returns the **current system date and time**.

**Example:**

```sql
SELECT GETDATE();    -- SQL Server
SELECT NOW();        -- MySQL, PostgreSQL
```

**Output Example:**
`2025-10-10 14:45:32`

---

## ‚è≥ **10. EXTRACT() Function (ANSI Standard / PostgreSQL)**

**Purpose:**
Extracts a specific part of a date/time ‚Äî similar to `DATEPART()`.

**Syntax:**

```sql
EXTRACT(field FROM source)
```

**Example:**

```sql
SELECT EXTRACT(YEAR FROM '2025-10-10'::DATE) AS year;
SELECT EXTRACT(MONTH FROM CURRENT_TIMESTAMP) AS month;
```

---

## ‚åõ **11. SYSDATE / CURRENT_DATE / CURRENT_TIMESTAMP**

**Purpose:**
System functions for current date/time values.

| Function            | Description                        | Example Output        |
| ------------------- | ---------------------------------- | --------------------- |
| `SYSDATE()`         | System‚Äôs current datetime (Oracle) | `2025-10-10 14:50:20` |
| `CURRENT_DATE`      | Current date only                  | `2025-10-10`          |
| `CURRENT_TIMESTAMP` | Current date and time              | `2025-10-10 14:50:20` |

---

## ‚è± **12. DATE_FORMAT() (MySQL)**

**Purpose:**
Formats date/time using custom format specifiers.

**Example:**

```sql
SELECT DATE_FORMAT('2025-10-10', '%W, %M %e, %Y');
```

**Result:**
`Friday, October 10, 2025`

---

## üîÅ **13. TIME_TO_SEC() and SEC_TO_TIME() (MySQL)**

**Purpose:**
Convert between time and seconds.

**Examples:**

```sql
SELECT TIME_TO_SEC('02:30:00');   -- 9000 seconds
SELECT SEC_TO_TIME(3600);         -- '01:00:00'
```

---

## üßÆ **14. ADDDATE() / SUBDATE()**

**Purpose:**
Add or subtract days from a date.

**Example:**

```sql
SELECT ADDDATE('2025-10-10', INTERVAL 5 DAY);   -- 2025-10-15
SELECT SUBDATE('2025-10-10', INTERVAL 3 DAY);   -- 2025-10-07
```

---

## üïï **15. CURDATE() / CURTIME()**

**Purpose:**
Return current date and time separately.

**Example:**

```sql
SELECT CURDATE();   -- 2025-10-10
SELECT CURTIME();   -- 14:52:10
```
