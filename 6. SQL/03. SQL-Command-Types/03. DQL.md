## üß© **Introduction**

**Data Query Language (DQL)** is a subset of **SQL (Structured Query Language)** used to **retrieve and view data** stored in database tables.
It focuses **only on querying** (reading) data ‚Äî without altering or deleting it.

DQL is the most frequently used part of SQL because almost every database operation involves **fetching data for display, reporting, or analysis**.

## üéØ **Purpose**

-   To **retrieve data** from the database.
-   To **analyze, filter, and display** information stored in one or more tables.
-   To **combine data** from multiple tables using joins and subqueries.

---

## üß± **Main DQL Command**

### üü© **SELECT ‚Äì Retrieve Data**

The `SELECT` statement is the **only** command in DQL.
It retrieves specific data from one or more tables based on given conditions.

### üßæ **Basic Syntax:**

```sql
SELECT column1, column2, ...
FROM table_name
WHERE condition
ORDER BY column
GROUP BY column
HAVING condition;
```

Each part of the query serves a specific function:

| Clause       | Function                                          |
| ------------ | ------------------------------------------------- |
| **SELECT**   | Specifies the columns to retrieve                 |
| **FROM**     | Specifies the table(s) to query                   |
| **WHERE**    | Filters rows based on a condition                 |
| **ORDER BY** | Sorts the output in ascending or descending order |
| **GROUP BY** | Groups rows for aggregate calculations            |
| **HAVING**   | Filters groups based on aggregate conditions      |

## üß† **Examples**

### 1Ô∏è‚É£ **Retrieve Specific Columns**

```sql

```

üìç _Fetches only the `name` and `department` columns from the `employees` table._

### 2Ô∏è‚É£ **Retrieve with Condition**

```sql

```

üìç _Retrieves names and departments of employees whose salary is greater than 50,000._

### 3Ô∏è‚É£ **Sort the Results**

```sql

```

üìç _Displays employee names and salaries, sorted from highest to lowest salary._

### 4Ô∏è‚É£ **Aggregate Data**

```sql

FROM employees
GROUP BY department;
```

üìç _Shows the average salary for each department._

### 5Ô∏è‚É£ **Filter Groups (Using HAVING)**

```sql

FROM employees
GROUP BY department
HAVING COUNT(*) > 5;
```

üìç _Displays departments with more than 5 employees._

### 6Ô∏è‚É£ **Using Aliases for Readability**

```sql

```

üìç _Uses aliases to rename columns in the result set._

### 7Ô∏è‚É£ **Retrieve All Columns**


üìç _Retrieves all columns from the `employees` table._

## ‚öñÔ∏è **Difference Between DQL and DML**

| Basis                   | DQL (Data Query Language)      | DML (Data Manipulation Language) |
| ----------------------- | ------------------------------ | -------------------------------- |
| **Purpose**             | Retrieve (read) data           | Insert, update, or delete data   |
| **Command Used**        | SELECT                         | INSERT, UPDATE, DELETE           |
| **Effect on Data**      | Does not change data           | Modifies data                    |
| **Transaction Control** | No need for commit or rollback | Requires commit or rollback      |
| **Operation Type**      | Read-only                      | Read and write                   |
