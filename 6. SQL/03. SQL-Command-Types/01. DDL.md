# **Data Definition Language (DDL)**

**Data Definition Language (DDL)** is a subset of **SQL (Structured Query Language)** that deals with **defining, altering, and managing the structure** of database objects ‚Äî not the data itself.

It defines _what the database looks like_ ‚Äî its **schema**, **tables**, **views**, **indexes**, and **constraints**.
DDL statements are mainly used by **database administrators (DBAs)** and **developers** during the design and maintenance phase of a database.

---

## ‚öôÔ∏è **Key Characteristics of DDL**

| Feature              | Description                                                                     |
| -------------------- | ------------------------------------------------------------------------------- |
| **Purpose**          | Defines and manages database structure (schema, tables, views, etc.)            |
| **Effect**           | Makes permanent structural changes in the database                              |
| **Automatic Commit** | DDL statements are auto-committed ‚Äî changes cannot be rolled back once executed |
| **Affects Schema**   | Modifies the database schema and not the actual data                            |
| **User Level**       | Typically used by database administrators or schema designers                   |

---

## üß± **Main DDL Commands**

### 1. **CREATE**

The `CREATE` command is used to **create new database objects**, such as tables, databases, views, or indexes.

#### üßæ **Syntax:**

```sql
CREATE TABLE table_name (
    column1 datatype constraints,
    column2 datatype constraints,
    ...
);
```

#### üß† **Example:**

```sql
CREATE TABLE Student (
    student_id INT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    age INT,
    course VARCHAR(50)
);
```

üìç _This command creates a new table named `Student` with four columns._

---

### 2. **ALTER**

The `ALTER` command is used to **modify an existing database object** ‚Äî such as adding, deleting, or changing a column in a table.

#### üßæ **Syntax:**

```sql
ALTER TABLE table_name
ADD column_name datatype;

ALTER TABLE table_name
DROP COLUMN column_name;

ALTER TABLE table_name
MODIFY column_name datatype;
```

#### üß† **Example:**

```sql
ALTER TABLE Student ADD email VARCHAR(100);
ALTER TABLE Student MODIFY age SMALLINT;
ALTER TABLE Student DROP COLUMN course;
```

üìç _This adds an `email` column, modifies the `age` column type, and removes the `course` column._

---

### 3. **DROP**

The `DROP` command **deletes an existing database object permanently**, including all data and structure.

#### üßæ **Syntax:**

```sql
DROP TABLE table_name;
DROP DATABASE database_name;
```

#### üß† **Example:**

```sql
DROP TABLE Student;
```

üìç _This removes the `Student` table completely from the database. Once dropped, it cannot be recovered._

---

### 4. **TRUNCATE**

The `TRUNCATE` command **removes all records from a table**, but **preserves the structure** (the table definition remains intact).

#### üßæ **Syntax:**

```sql
TRUNCATE TABLE table_name;
```

#### üß† **Example:**

```sql
TRUNCATE TABLE Student;
```

üìç _This deletes all data from the `Student` table but keeps the table ready for future use._

üß© **Note:**
`TRUNCATE` is faster than `DELETE` because it does not generate individual row delete operations or transaction logs.

---

### 5. **RENAME** (Optional command in some RDBMS)

The `RENAME` command is used to **change the name** of an existing database object like a table.

#### üßæ **Syntax:**

```sql
RENAME TABLE old_name TO new_name;
```

#### üß† **Example:**

```sql
RENAME TABLE Student TO Learner;
```

üìç _This changes the table name from `Student` to `Learner`._

---

## üß† **Difference Between DDL and DML**

| Basis           | DDL (Data Definition Language)        | DML (Data Manipulation Language) |
| --------------- | ------------------------------------- | -------------------------------- |
| **Purpose**     | Defines the structure of the database | Manages the data within tables   |
| **Examples**    | CREATE, ALTER, DROP, TRUNCATE         | SELECT, INSERT, UPDATE, DELETE   |
| **Affects**     | Schema (metadata)                     | Actual data                      |
| **Auto Commit** | Yes                                   | No (requires manual COMMIT)      |
| **Rollback**    | Not possible                          | Possible                         |

---

## üß© **How DDL Affects Database Schema**

-   Defines relationships between tables using **constraints** (PRIMARY KEY, FOREIGN KEY, UNIQUE, etc.)
-   Controls **data integrity** through schema-level rules
-   Defines **indexes** to improve query performance
-   Establishes **views** for controlled access and abstraction

## ‚úÖ **Summary**

| Command      | Function                     | Description                                  |
| ------------ | ---------------------------- | -------------------------------------------- |
| **CREATE**   | Create a new database object | Defines tables, views, indexes, etc.         |
| **ALTER**    | Modify structure             | Add/remove/modify columns                    |
| **DROP**     | Delete structure             | Removes table or database permanently        |
| **TRUNCATE** | Delete all data              | Keeps structure but removes all rows         |
| **RENAME**   | Rename object                | Changes table or column name (in some RDBMS) |

## üîó **Real-World Use Cases**

-   Setting up database schema during **initial design phase**
-   Modifying schema during **database migration or version updates**
-   Clearing test data using `TRUNCATE` during **development/testing**
-   Removing outdated tables or views using `DROP`
-   Enforcing **data integrity and normalization** through DDL constraints
