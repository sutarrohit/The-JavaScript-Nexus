The `position` utilities control **how an element is positioned within the document layout**.
They define whether an element participates in the **normal document flow** or is **taken out of it**, and **how its offsets (top, right, bottom, left)** are calculated relative to other elements.

These utilities are key to **layout control**, **overlays**, **sticky headers**, **tooltips**, and **custom UI components**.

## ğŸ¨ **Available Classes and Their Meanings**

| **Class**  | **CSS Property**      | **Description**                                                                            |

# 2. ğŸ§© **What are `top`, `right`, `bottom`, `left` & `inset` in Tailwind CSS?**

These utilities control the **placement and offset** of **positioned elements** (i.e., elements with `relative`, `absolute`, `fixed`, or `sticky` positioning).

They define **how far** an element is positioned from one or more sides of its containing block (parent or viewport).

## ğŸ¨ **Available Classes and Their Meanings**

| **Class**     | **CSS Property**                                      | **Description**                           |
| ------------- | ----------------------------------------------------- | ----------------------------------------- |

# 4. ğŸ§© **What is `z-index` in Tailwind CSS?**

The `z-index` utilities control the **stacking order** of elements along the **z-axis** â€” the imaginary line that runs perpendicular to the screen.
This determines **which elements appear on top of others** when they overlap.

Elements with higher `z-index` values are rendered **above** elements with lower values â€” but **only within the same stacking context**.
