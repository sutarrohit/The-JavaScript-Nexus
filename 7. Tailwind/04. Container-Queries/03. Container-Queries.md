Responsive design ensures that your website **adapts to different screen sizes** â€” like mobile phones, tablets, and desktops.

## ğŸ“± **4. Mobile-First Approach**

Tailwind is **mobile-first**.
That means:

-   **Unprefixed** classes apply to **all screen sizes**.
-   **Prefixed** classes (like `md:`) apply **only at and above** that breakpoint.

ğŸ“˜ **Example:**

```html
<!-- Wrong: This centers text only on â‰¥640px -->
<div class="sm:text-center"></div>

<!-- Correct: Center on mobile, left-align on sm and above -->
<div class="text-center sm:text-left"></div>
```

ğŸ§  **Tip:**
Start with mobile layout first (base classes), then layer on larger breakpoints like `sm:`, `md:`, `lg:` progressively.

## ğŸ¯ **5. Targeting Breakpoint Ranges**

Normally, `md:flex` means the style applies **at `md` and above**.
But sometimes you want it **only between certain screen sizes**.

You can **stack variants** to target a specific range.

ğŸ“˜ **Example:**

```html
<!-- Apply flex only between md and xl -->

```

| Variant   | Media Query    |
| --------- | -------------- |
| `max-sm`  | width < 640px  |
| `max-md`  | width < 768px  |
| `max-lg`  | width < 1024px |
| `max-xl`  | width < 1280px |
| `max-2xl` | width < 1536px |

ğŸ“˜ **Single Breakpoint Example:**

```html

```

## ğŸ§¹ **7. Removing Default Breakpoints**

ğŸ“˜ **Example:**

## ğŸ§¾ **Summary**

| Concept                   | Description                                                             |
| ------------------------- | ----------------------------------------------------------------------- |
| **Responsive Variants**   | Apply styles conditionally based on screen width.                       |

| **Container Queries**     | Style components based on parent size instead of viewport.              |
| **Container Units (cqw)** | Use container-relative units for sizing.                                |
