## üß© Overview

These **iteration helper methods** return **iterators**, not arrays.
They‚Äôre part of the **Iterable protocol**, introduced in **ES6 (ECMAScript 2015)** ‚Äî which means you can use them directly in `for...of` loops, the spread operator (`...`), and other iterable contexts.

| Method      | What it returns        | Commonly used with                 |
| ----------- | ---------------------- | ---------------------------------- |
| `entries()` | `[index, value]` pairs | When you need both index and value |
| `keys()`    | indices only           | When you only need array indices   |
| `values()`  | values only            | When you only need elements        |

## üîπ 1. `entries()`

### üß† Purpose:

Returns a **new iterator** that gives you **[index, value] pairs** for each element in the array.

### üß© Syntax:

```js
array.entries();
```

### üß™ Example:

```js
const fruits = ["apple", "banana", "cherry"];

for (const [index, value] of fruits.entries()) {
    console.log(index, value);
}
```

**Output:**

```
0 apple
1 banana
2 cherry
```

### üí° Use cases:

-   When you need both the index **and** the element while looping.
-   Better alternative to `forEach((v, i) => ‚Ä¶)` if you prefer `for...of` syntax.
-   Often used in functional loops, React key mapping, or data logging.

### ‚öôÔ∏è Convert iterator to array:

```js
console.log([...fruits.entries()]);
// [[0, "apple"], [1, "banana"], [2, "cherry"]]
```

---

## üîπ 2. `keys()`

### üß† Purpose:

Returns an **iterator** containing the **indices (keys)** of the array.

### üß© Syntax:

```js
array.keys();
```

### üß™ Example:

```js
const fruits = ["apple", "banana", "cherry"];

for (const i of fruits.keys()) {
    console.log(i);
}
```

**Output:**

```
0
1
2
```

### üí° Use cases:

-   When you only care about **indexes**.
-   Useful for manual index-based operations, e.g., matrix traversal, double loops, or custom algorithms.

### ‚öôÔ∏è Convert iterator to array:

```js
console.log([...fruits.keys()]);
// [0, 1, 2]
```

---

## üîπ 3. `values()`

### üß† Purpose:

Returns an **iterator** that yields only the **values** of the array.

### üß© Syntax:

```js
array.values();
```

### üß™ Example:

```js
const fruits = ["apple", "banana", "cherry"];

for (const value of fruits.values()) {
    console.log(value);
}
```

**Output:**

```
apple
banana
cherry
```

### üí° Use cases:

-   Ideal for clean iteration when you **only need values** (similar to `for...of array` directly).
-   Useful for generators, lazy evaluation, and working with `Map`/`Set` structures that return iterators too.

### ‚öôÔ∏è Convert iterator to array:

```js
console.log([...fruits.values()]);
// ["apple", "banana", "cherry"]
```

## üîÑ Comparison with other loops

| Method                     | Returns               | Equivalent                                                       |
| -------------------------- | --------------------- | ---------------------------------------------------------------- |
| `for...of`                 | values                | same as `arr.values()`                                           |
| `for...in`                 | **keys (as strings)** | like `arr.keys()` but includes inherited props ‚Äî not recommended |
| `arr.forEach((v, i) => ‚Ä¶)` | values via callback   | but doesn‚Äôt return iterator                                      |

---

## ‚ö†Ô∏è Important Notes

-   These methods **don‚Äôt mutate** the array (‚úÖ non-mutating).
-   They return **iterators**, which are _consumed once_.
-   You can convert iterators into arrays using `[...iterator]` or `Array.from()`.
