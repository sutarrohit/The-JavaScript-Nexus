## üß© 1. `Array.isArray()`

### üß† Purpose:

Checks whether a given value is an **array**.

### üß© Syntax:

```js
Array.isArray(value);
```

### üß™ Example:

```js
Array.isArray([1, 2, 3]); // true
Array.isArray("hello"); // false
Array.isArray({ length: 3 }); // false
Array.isArray(new Array()); // true
```

### üí° Why it‚Äôs useful:

-   `typeof []` returns `"object"` ‚Äî so `Array.isArray()` is the **correct way** to check for arrays.
-   Especially important when dealing with data from APIs or libraries.

‚úÖ **Does not mutate** anything.
‚úÖ **Static method** ‚Üí called directly on `Array`, not on an array instance.

## üß© 2. `Array.fromAsync()` (üîπ _ES2024 ‚Äì New!_)

### üß† Purpose:

An **asynchronous version of `Array.from()`** ‚Äî it converts **async iterables, promises, or async generators** into arrays.

### üß© Syntax:

```js
await Array.fromAsync(asyncIterable [, mapFn])
```

### üß™ Example 1 ‚Äì Async generator

```js
async function* asyncGen() {
    yield 1;
    yield 2;
    yield 3;
}

const arr = await Array.fromAsync(asyncGen());
console.log(arr); // [1, 2, 3]
```

### üß™ Example 2 ‚Äì With mapping function

```js
async function* asyncGen() {
    yield 2;
    yield 4;
    yield 6;
}

const result = await Array.fromAsync(asyncGen(), (x) => x * 2);
console.log(result); // [4, 8, 12]
```

### üí° Why it‚Äôs powerful:

-   Handles **asynchronous sources** like streams or network data.
-   Can replace manual `for await...of` loops when you need the results in an array.

‚úÖ Returns a **Promise** that resolves to an array.
‚úÖ Works with **async iterators**, **Promises**, or **async generators**.

## üß© 3. `Array.prototype.toReversed()`

### üß† Purpose:

Returns a **reversed copy** of the array **without mutating** the original.

### üß© Syntax:

```js
const newArr = arr.toReversed();
```

### üß™ Example:

```js
const arr = [1, 2, 3];
const rev = arr.toReversed();

console.log(rev); // [3, 2, 1]
console.log(arr); // [1, 2, 3]  ‚Üê original not changed
```

### üí° Why it‚Äôs useful:

-   Normal `arr.reverse()` **mutates** the array.
-   `toReversed()` is **immutable** ‚Äî perfect for React state, functional programming, or safe data handling.

‚úÖ **Non-mutating**
‚úÖ Introduced in **ES2023**

## üß© 4. `Array.prototype.toSorted(compareFn)`

### üß† Purpose:

Creates a **sorted copy** of the array **without modifying** the original.

### üß© Syntax:

```js
const sorted = arr.toSorted((a, b) => a - b);
```

### üß™ Example:

```js
const nums = [3, 1, 2];
const sorted = nums.toSorted((a, b) => a - b);

console.log(sorted); // [1, 2, 3]
console.log(nums); // [3, 1, 2]
```

### üí° Why it‚Äôs useful:

-   Normal `arr.sort()` **mutates** arrays.
-   `toSorted()` is the **immutable** version ‚Äî perfect for pure functions and safe operations.

‚úÖ Non-mutating
‚úÖ Uses same compare rules as `sort()`

## üß© 5. `Array.prototype.toSpliced(start, deleteCount, ...items)`

### üß† Purpose:

Immutable version of `splice()` ‚Äî returns a **new array** instead of changing the old one.

### üß© Syntax:

```js
const newArr = arr.toSpliced(start, deleteCount, ...items);
```

### üß™ Example:

```js
const arr = [1, 2, 3, 4];
const spliced = arr.toSpliced(1, 2, "X", "Y");

console.log(spliced); // [1, "X", "Y", 4]
console.log(arr); // [1, 2, 3, 4]
```

### üí° Why it‚Äôs useful:

-   Normal `splice()` removes/adds items **in place**.
-   `toSpliced()` returns a **new modified copy**.
-   Great for React state updates or immutable data flows.

‚úÖ Non-mutating
‚úÖ Same arguments as `splice()`

## üß© 6. `Array.prototype.with(index, value)`

### üß† Purpose:

Replaces an element at a specific index with a new value ‚Äî **without mutating** the array.

### üß© Syntax:

```js
const newArr = arr.with(index, newValue);
```

### üß™ Example:

```js
const arr = ["a", "b", "c"];
const newArr = arr.with(1, "z");

console.log(newArr); // ["a", "z", "c"]
console.log(arr); // ["a", "b", "c"]
```

### üí° Why it‚Äôs useful:

-   Immutable alternative to `arr[index] = value`.
-   Perfect for updating arrays in **React state** or **functional style**.

‚úÖ Non-mutating
‚úÖ Throws error if index is out of range
‚úÖ Works like a ‚Äúsafe replace‚Äù

## üß† Why These Matter

### üöÄ Immutability by Design

Older array methods like `sort()`, `reverse()`, `splice()` mutate arrays, which can lead to **bugs** in:

-   Functional programming
-   React or Vue (state immutability)
-   Redux reducers

The new methods (`toSorted`, `toReversed`, `toSpliced`, `with`) **always return new arrays** ‚Äî no side effects.

### ‚ö° Static Helpers

Methods like `Array.isArray()` and `Array.fromAsync()` help with **type checking** and **async iteration**, expanding JavaScript‚Äôs capability to handle more complex data sources safely.
