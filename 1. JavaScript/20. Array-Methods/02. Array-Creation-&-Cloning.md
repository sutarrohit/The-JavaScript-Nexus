## ğŸ§± 2. `Array.of()`

**Purpose:**
Creates an array from **individual arguments**, not from iterables.

**Syntax:**

```js
Array.of(element1, element2, ...)
```

**Details:**

-   Always creates an array with the **exact arguments** passed.
-   Differs from `Array()` constructor, which behaves oddly with a single numeric argument.

**Examples:**

```js

// â†’ [1, 2, 3]

// Difference from Array()

// â†’ [ <3 empty items> ]

Array.of(3);
// â†’ [3]
```

**Use case:**
âœ… When you need to **safely create an array** from values without worrying about special behavior of `Array()`.

**Purpose:**

**Syntax:**

```js

```

**Details:**

-   Returns a **new array** without modifying the original.
-   Works with **positive** and **negative** indices.
-   Often used to **clone arrays** or extract subarrays immutably.

**Examples:**

```js

// Clone entire array

// â†’ [1, 2, 3, 4, 5]

// â†’ [4, 5]
```

**Use case:**

**Purpose:**
Merges arrays or values **without mutating** the originals.

**Syntax:**

```js

```

**Details:**

-   Returns a **new array**.
-   Can also concatenate non-array values (numbers, strings, etc.).

**Examples:**

```js

// â†’ [1, 2, 3, 4]

// â†’ [1, 2, 5, 6, 7]

// a remains unchanged

// â†’ [1, 2]
```

**Use case:**
âœ… When you need to **combine multiple arrays** immutably (e.g., merging results or building new datasets).
