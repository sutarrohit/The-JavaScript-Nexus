## **1. What is `useRef()`**

`useRef` is a built-in React Hook that **creates a mutable reference** to store values that **persist between renders** but **do not trigger re-renders** when updated.

It is commonly used for:

- **Accessing the DOM** (e.g., focusing an input field)
- **Persisting values across renders** (without causing re-renders)
- **Storing previous state values**

---

## **1ï¸âƒ£ Syntax of `useRef`**

```jsx
const ref = useRef(initialValue);
```

- `ref.current` holds the reference value.
- The value persists across renders.
- Updating `ref.current` does **not** trigger a re-render.

---

## **2ï¸âƒ£ `useRef` for Accessing the DOM**

One of the main uses of `useRef` is to get direct access to a **DOM element**.

### **ğŸ“Œ Example: Auto-Focus on Input Field**

```jsx
import React, { useRef, useEffect } from "react";

function AutoFocusInput() {
  const inputRef = useRef(null); // Creating a reference

  useEffect(() => {
    inputRef.current.focus(); // Focus input field when the component mounts
  }, []);

  return <input ref={inputRef} type="text" placeholder="Type here..." />;
}

export default AutoFocusInput;
```

ğŸ”¹ **How it works?**

- `useRef(null)` creates a reference.
- `ref={inputRef}` attaches it to the `<input>` element.
- `inputRef.current.focus()` **automatically focuses the input** on mount.

---

## **3ï¸âƒ£ `useRef` for Storing Values Without Re-Renders**

Unlike `useState`, updating `useRef` does **not** cause a re-render.

### **ğŸ“Œ Example: Tracking Render Count**

```jsx
import React, { useRef, useState, useEffect } from "react";

function RenderCounter() {
  const [count, setCount] = useState(0);
  const renderCount = useRef(1); // Initialize with 1

  useEffect(() => {
    renderCount.current += 1; // Increment on every render
  });

  return (
    <div>
      <h2>Count: {count}</h2>
      <h3>Component Rendered: {renderCount.current} times</h3>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default RenderCounter;
```

ğŸ”¹ **Why use `useRef` instead of `useState`?**

- If we used `useState`, every update would **trigger a re-render**, creating an infinite loop.
- With `useRef`, **the value persists across renders without causing extra re-renders**.

## **4ï¸âƒ£ `useRef` for Storing Previous State**

We can use `useRef` to **store the previous state value** for comparison.

### **ğŸ“Œ Example: Tracking Previous Count Value**

```jsx

  const [count, setCount] = useState(0);

    prevCount.current = count; // Update after each render
  }, [count]);

  return (
    <div>
      <h2>Current Count: {count}</h2>
      <h3>Previous Count: {prevCount.current}</h3>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default PreviousStateExample;
```

ğŸ”¹ **Why is `useRef` useful here?**

- `prevCount.current` holds the **previous state value**.
- It **persists across renders** without causing additional re-renders.

## **6ï¸âƒ£ `useRef` for Handling Timers**

`useRef` is useful for managing **setTimeout** or **setInterval** without causing re-renders.

### **ğŸ“Œ Example: Stopping a Timer**

```jsx
import React, { useState, useRef } from "react";

function Timer() {
  const [count, setCount] = useState(0);
  const timerRef = useRef(null); // Store timer ID

  const startTimer = () => {
    timerRef.current = setInterval(() => {
      setCount((prevCount) => prevCount + 1);
    }, 1000);
  };

  const stopTimer = () => {
    clearInterval(timerRef.current); // Clear interval
  };

  return (
    <div>
      <h2>Timer: {count}</h2>
      <button onClick={startTimer}>Start</button>
      <button onClick={stopTimer}>Stop</button>
    </div>
  );
}

export default Timer;
```

ğŸ”¹ **Why use `useRef`?**

- **Stores the timer ID** without causing re-renders.
- Using `useState` would cause unnecessary re-renders, slowing down performance.

## **7ï¸âƒ£ `useRef` in Event Handlers**

Since `useRef` **does not change between renders**, it is useful in event handlers.

### **ğŸ“Œ Example: Tracking Click Count Without Re-renders**

```jsx

  const clickCount = useRef(0);

  const handleClick = () => {
    clickCount.current += 1;
    console.log(`Button clicked ${clickCount.current} times`);
  };

}
```

ğŸ”¹ **Why use `useRef`?**

- If `useState` was used, clicking the button **would re-render the component every time**.
- `useRef` updates the value **without re-rendering**, making it more efficient.

## **8ï¸âƒ£ When to Use `useRef`?**

| **Use Case**                          | **Why `useRef`?**                                           |
| ------------------------------------- | ----------------------------------------------------------- |
| **Accessing the DOM**                 | Get references to elements (e.g., auto-focus input).        |
| **Storing values without re-renders** | Maintain data across renders (e.g., render count).          |
| **Tracking previous state**           | Store past values without affecting renders.                |

---

## **9ï¸âƒ£ Best Practices for `useRef`**

âœ… **Use `useRef` when you need a persistent value that doesn't trigger re-renders.**  
âœ… **Use `useRef` for accessing the DOM directly (like input focus, measuring elements).**  
âœ… **Avoid using `useRef` for updating UI state (use `useState` for that instead).**  
âœ… **When tracking state changes, combine `useRef` with `useEffect`.**  
âœ… **Do not update `ref.current` inside the render function**â€”it wonâ€™t take effect until the next render.

---

## **ğŸ”¹ Summary**

âœ” **`useRef` creates a mutable reference** that persists between renders but **does not cause re-renders**.

## **2. What is `useReducer()`**

`useReducer` is an alternative to `useState` for **managing complex state logic** in React. It is especially useful when:

- The next state **depends on the previous state**.
- The state **transitions** involve multiple conditions.
- **Centralized state management** is required.

ğŸ‘‰ `useReducer` is similar to Redux, as it follows the **"reducer pattern"** where **state is updated based on dispatched actions**.

---

## **1ï¸âƒ£ Syntax of `useReducer`**

```jsx
const [state, dispatch] = useReducer(reducer, initialState);
```

- **`reducer`** â†’ A function that takes `state` and `action`, then returns a **new state**.
- **`initialState`** â†’ The starting value of the state.
- **`dispatch`** â†’ A function used to send actions that update the state.

---

## **2ï¸âƒ£ Basic Example: Counter with `useReducer`**

Let's implement a simple counter that can **increment, decrement, and reset**.

### **ğŸ“Œ Step 1: Define Reducer Function**

```jsx
function counterReducer(state, action) {
  switch (action.type) {
    case "INCREMENT":
      return { count: state.count + 1 };
    case "DECREMENT":
      return { count: state.count - 1 };
    case "RESET":
      return { count: 0 };
    default:
      return state;
  }
}
```

ğŸ”¹ The **reducer function** takes `state` and `action` and returns the **new state** based on `action.type`.

---

### **ğŸ“Œ Step 2: Use `useReducer` in a Component**

```jsx
import React, { useReducer } from "react";

function Counter() {
  const [state, dispatch] = useReducer(counterReducer, { count: 0 });

  return (
    <div>
      <h2>Count: {state.count}</h2>
      <button onClick={() => dispatch({ type: "INCREMENT" })}>+</button>
      <button onClick={() => dispatch({ type: "DECREMENT" })}>-</button>
      <button onClick={() => dispatch({ type: "RESET" })}>Reset</button>
    </div>
  );
}

export default Counter;
```

ğŸ”¹ The `dispatch` function **triggers actions**, updating the state **based on the reducer function**.

---

## **3ï¸âƒ£ `useReducer` vs `useState`**

| Feature                             | `useState`                                | `useReducer`                                       |
| ----------------------------------- | ----------------------------------------- | -------------------------------------------------- |
| **Best for**                        | Simple state updates                      | Complex state logic                                |
| **Updates based on previous state** | âŒ No, manual updates needed              | âœ… Yes, reducer handles state transitions          |
| **Manages multiple state values**   | âŒ No, requires multiple `useState` calls | âœ… Yes, can handle multiple state changes together |
| **Centralized state logic**         | âŒ No                                     | âœ… Yes                                             |
| **Performance**                     | âœ… Good for small state updates           | âœ… Better for large, complex state                 |

ğŸ‘‰ **Use `useState` for simple state updates, and `useReducer` for complex state transitions.**

## **4ï¸âƒ£ Using `useReducer` with Multiple State Properties**

Unlike `useState`, which requires multiple variables, `useReducer` can **manage multiple state values in a single object**.

### **ğŸ“Œ Example: Managing Form State**

```jsx

      return { ...state, name: action.payload };

      return { ...state, email: action.payload };
    case "RESET":
      return { name: "", email: "" };
    default:
      return state;
  }

```

```jsx
  return (
    <div>
      <input
        type="text"
        value={state.name}
        onChange={(e) =>
          dispatch({ type: "UPDATE_NAME", payload: e.target.value })
        }
        placeholder="Enter Name"
      />
      <input
        type="email"
        value={state.email}
        onChange={(e) =>
          dispatch({ type: "UPDATE_EMAIL", payload: e.target.value })
        }
        placeholder="Enter Email"
      />
      <button onClick={() => dispatch({ type: "RESET" })}>Reset</button>
      <h3>Name: {state.name}</h3>
      <h3>Email: {state.email}</h3>
    </div>
  );
}
```

ğŸ”¹ **Advantages**:  
âœ… Manages multiple inputs in **a single state object**.  
âœ… **No need for multiple `useState` calls**.

## **3. What is `useContext()`**

The **`useContext` Hook** allows components to **access shared data (global state) without passing props manually through every level of the component tree**.

This solves the problem of **"prop drilling"**, where deeply nested components need access to the same data but have to receive it via multiple intermediate parent components.

## **1ï¸âƒ£ Why Use `useContext`?**

âœ… **Avoids prop drilling** â†’ No need to pass props manually through multiple components.  
âœ… **Simplifies global state management** â†’ Easily share data like user authentication, theme, language, etc.  
âœ… **Works well with `useState` and `useReducer`** for complex state management.

## **2ï¸âƒ£ Syntax of `useContext`**

```jsx
const contextValue = useContext(MyContext);
```

- `MyContext` is a **context object** created using `React.createContext()`.
- `useContext(MyContext)` returns the **current context value**.
- The component **re-renders** when the context value changes.

---

## **3ï¸âƒ£ Basic Example: Theme Context**

Let's create a **Theme Context** that allows components to switch between **light** and **dark** themes.

### **ğŸ“Œ Step 1: Create Context**

```jsx

  const [theme, setTheme] = useState("light");

  const toggleTheme = () => {
    setTheme(theme === "light" ? "dark" : "light");
  };

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  );
}

export default ThemeContext;
```

ğŸ”¹ **`ThemeContext.Provider`** makes `theme` and `toggleTheme` available to child components.

---

```jsx
    </div>
  );

```
