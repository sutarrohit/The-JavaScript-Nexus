# 1. **Mapped Types**

**Mapped Types** in TypeScript allow you to create new types **by transforming the properties** of an existing type. They are useful when you need to modify all properties of a type in a structured way, such as making them optional, readonly, or converting their types.

## **1Ô∏è‚É£ Understanding Mapped Types**

Mapped types use the `keyof` operator to iterate over the properties of an existing type and apply transformations.

### **Basic Syntax:**

```ts
type NewType<T> = {
  [K in keyof T]: TypeTransformation;
};
```

- `T` is the original type.
- `keyof T` extracts the keys of `T`.
- `K in keyof T` iterates over each key in `T`.
- `TypeTransformation` defines how the property should be transformed.

---

## **2Ô∏è‚É£ Example: Making Properties Optional (`?`)**

You can use mapped types to **make all properties optional**.

### **Before Using Mapped Types**

```ts
type User = {
  name: string;
  age: number;
};
```

You would typically write:

```ts
type PartialUser = {
  name?: string;
  age?: number;
};
```

But with mapped types, you can automate this:

### **Using Mapped Types**

```ts
type Partial<T> = {
  [K in keyof T]?: T[K];
};

type PartialUser = Partial<User>;
```

‚úÖ **Now, all properties of `User` are optional:**

```ts
const user: PartialUser = {
  name: "Alice", // ‚úÖ Valid, 'age' is optional
};
```

---

## **3Ô∏è‚É£ Example: Making Properties Read-Only (`readonly`)**

You can make all properties **read-only** to prevent modification.

### **Using Mapped Types**

```ts
type Readonly<T> = {
  readonly [K in keyof T]: T[K];
};

type ReadonlyUser = Readonly<User>;
```

‚úÖ **Now, modifying `ReadonlyUser` properties will cause an error:**

```ts
const user: ReadonlyUser = { name: "Bob", age: 25 };

// user.name = "Alice"; ‚ùå Error: Cannot assign to 'name' because it is a read-only property.
```

---

## **4Ô∏è‚É£ Example: Removing `readonly` and `?` Modifiers**

If you want to **remove `readonly` and optional (`?`) modifiers**, you can use the `-readonly` and `-?` operators.

### **Example: Removing `readonly`**

```ts
type Mutable<T> = {
  -readonly [K in keyof T]: T[K];
};

type NormalUser = Mutable<ReadonlyUser>;
```

‚úÖ **Now, `NormalUser` properties are no longer `readonly`.**

### **Example: Removing Optional (`?`)**

```ts
type Required<T> = {
  [K in keyof T]-?: T[K];
};

type RequiredUser = Required<PartialUser>;
```

‚úÖ **Now, `RequiredUser` properties are mandatory.**

---

## **5Ô∏è‚É£ Example: Changing Property Types**

Mapped types can modify the **type of properties dynamically**.

### **Example: Convert Properties to Strings**

```ts
type Stringify<T> = {
  [K in keyof T]: string;
};

type StringUser = Stringify<User>;
```

‚úÖ **Now, all properties of `User` will be strings:**

```ts
const user: StringUser = { name: "Alice", age: "30" }; // ‚úÖ Only strings allowed
```

---

## **6Ô∏è‚É£ Example: Mapping Over Functions**

Mapped types can transform **function types**.

### **Example: Convert Functions to Return `Promise<T>`**

```ts
type Async<T> = {
  [K in keyof T]: () => Promise<T[K]>;
};

type UserAsync = Async<User>;
```

‚úÖ **Now, every function in `UserAsync` returns a promise:**

```ts
const user: UserAsync = {
  name: async () => "Alice",
  age: async () => 25,
};
```

---

## **7Ô∏è‚É£ Example: Filtering Properties Using `never`**

You can **remove properties** by setting them to `never`.

### **Example: Remove `age` Property**

```ts
type OmitAge<T> = {
  [K in keyof T as K extends "age" ? never : K]: T[K];
};

type UserWithoutAge = OmitAge<User>;
```

‚úÖ **Now, `UserWithoutAge` does not have an `age` property.**

```ts
const user: UserWithoutAge = { name: "Alice" }; // ‚úÖ Only 'name' remains
```

## **üöÄ Key Takeaways**

‚úÖ **Mapped types allow you to dynamically transform object types.**  
‚úÖ **They use `keyof` to iterate over object properties.**  
‚úÖ **You can make properties optional, readonly, or change their types.**  
‚úÖ **They help reduce code repetition and improve maintainability.**
